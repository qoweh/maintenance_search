# 데이터 분석 대회 수상작 기반 모델링 적용방안

작성일: 2026-06-11

## 검토 방향

유지보수 내역서는 정답 라벨이 없는 짧은 한국어 업무 로그 데이터다. 따라서 일반적인 대회 수상작처럼 단순히 분류 모델 성능을 높이는 접근보다는, 다음 4가지를 보고서용 분석 프레임으로 조합하는 것이 적합하다.

1. 텍스트 토픽모델링으로 장애유형을 자동 도출
2. 코사인 유사도 그래프로 유사·반복 장애군을 도출
3. 룰 마이닝으로 장애내용-조치내용 표준 대응 후보를 추출
4. 시계열 이상탐지로 특정 일자·월의 업무량 급증을 설명

## 수상작·공개 솔루션에서 가져올 만한 패턴

| 참고 영역 | 공개 자료에서 확인한 패턴 | 유지보수 데이터에 적용할 방법 |
|---|---|---|
| Kaggle 텍스트 분류 수상 솔루션 | Jigsaw, Quora 등 텍스트 대회 수상작은 전처리, 임베딩/TF-IDF, 딥러닝 또는 앙상블을 결합하는 흐름이 많음 | 정답 라벨이 없으므로 분류 대회 방식은 직접 적용보다 `텍스트 벡터화 + 유사도/군집화 + 모델 비교` 형태로 변환 |
| Kaggle 일반 수상 솔루션 | 공개 솔루션 모음과 메타 분석에서 교차검증, 앙상블, 피처 엔지니어링, 모델 비교가 반복적으로 등장 | 보고서에 모델별 비교표, 평가 지표, 최종 모델 선정 근거를 포함 |
| DACON 텍스트 분류 | 악성 URL 분류 사례에서 TF-IDF 벡터화, MLP, Dropout, EarlyStopping, ROC-AUC 모니터링 사용 | 유지보수 로그에는 `TF-IDF + NMF/KMeans/유사도`를 우선 적용하고, 라벨을 만들 경우 MLP/LightGBM 분류로 확장 |
| DACON 입법 데이터 시각화 | 법안 제목·내용 분석, 시계열 흐름, LDA, Spherical K-means 등 텍스트와 시간축 결합 | 장애유형별 월별 추이, Hot/Cold 장애유형, 부서별 토픽 분포 분석 |
| 국민권익위원회 민원 빅데이터 | 민원추이, 기관별 현황, 주요 키워드, 급증 민원, 이슈 분석을 운영 지표로 사용 | 유지보수 건수 추이, 부서별 현황, 주요 키워드, 급증 장애일 탐지로 구성 |
| 민원 토픽모델링 연구 | 부산시 민원 연구는 LDA로 20개 토픽을 추출하고 분기별 Hot/Cold 민원을 도출 | 유지보수 로그에도 토픽별 월별 비중을 산출해 Hot/Cold 장애유형 도출 |
| 2025 빅콘테스트 수상작 | RAG, 컨텍스트 엔지니어링, 군집화, 생존분석 기반 조기경보처럼 "예측 + 원인 진단" 구조 사용 | 반복 장애군과 표준조치 룰을 결합해 `장애 조기경보 및 대응 매뉴얼 후보`로 제안 |
| BERTopic 계열 | Transformer 임베딩, UMAP, HDBSCAN, c-TF-IDF로 해석 가능한 토픽 생성 | 고급모델 후보로 사용 가능. 다만 보고서 작성용 1차 분석은 경량 NMF가 더 안정적 |
| LightGBM + SHAP | 정형 피처 기반 예측 모델에서 변수 영향도를 설명하는 XAI 활용 | `고발생일 여부`, `반복장애 여부` 같은 라벨을 만들면 부서, 월, 유형, 키워드가 위험도에 미치는 영향을 설명 가능 |

## 이 데이터에 가장 잘 맞는 모델 조합

### 1순위: NMF 기반 장애유형 토픽모델

- 목적: 장애내용과 조치내용을 자동으로 8~12개 주제로 분해
- 이유: 유지보수 로그는 문장이 짧고 반복 표현이 많아, TF-IDF 기반 NMF가 LDA보다 해석이 선명함
- 보고서 표현: "TF-IDF 기반 비음수 행렬분해(NMF)를 적용하여 장애유형을 자동 분류하였다."
- 산출물: 토픽별 핵심어, 대표 사례, 월별 토픽 비중, 부서별 토픽 비중

### 2순위: 코사인 유사도 기반 반복 사례 그래프

- 목적: 동일하거나 거의 유사한 장애-조치 사례를 묶음
- 이유: 기존 정보공개청구 보고서의 유사·반복 청구 분석과 구조가 같다
- 보고서 표현: "장애내용과 조치내용의 TF-IDF 벡터 간 코사인 유사도를 계산하고, 임계값 이상인 사례를 연결하여 반복 장애군을 구성하였다."
- 산출물: 임계값별 반복 사례 수, 반복 그룹 수, 상위 반복 그룹, 평균 유사도

### 3순위: 장애내용-조치내용 룰 마이닝

- 목적: 특정 장애가 발생했을 때 자주 등장한 조치내용을 표준 대응 후보로 추출
- 이유: 보고서 제언이 단순 통계가 아니라 "실제 매뉴얼화"로 이어짐
- 보고서 표현: "장애내용을 조건부 항목, 조치내용을 결과 항목으로 두고 반복 빈도와 신뢰도를 산출하였다."
- 산출물: 표준조치 후보표, 빈도, 신뢰도, 대표 사례

### 4순위: 시계열 이상탐지

- 목적: 업무량이 비정상적으로 몰린 날짜와 월을 탐지
- 이유: 유지보수 운영상 인력 배치, 사전 점검, 시스템 업데이트 영향 설명 가능
- 보고서 표현: "일별 처리 건수의 평균과 표준편차를 기준으로 z-score를 산출하여 급증일을 탐지하였다."
- 산출물: 급증일 순위, z-score, 해당일 주요 장애유형

### 5순위: BERTopic 고급 토픽모델

- 목적: 단어가 달라도 의미가 가까운 사례를 묶음
- 이유: 대회·실무에서 최근 많이 쓰는 "그럴듯한" 모델명과 구조를 제시하기 좋음
- 주의: 보고서 작성 기간이 짧고 데이터가 2,639건이라, 최종 본문은 NMF를 주 모델로 쓰고 BERTopic은 보조·확장 모델로 두는 편이 안전함
- 보고서 표현: "Transformer 임베딩 기반 BERTopic을 보조 실험으로 검토하여 토픽 해석력을 비교하였다."

## 1차 모델 비교 결과

분석 대상은 2025.01~2026.04 유지보수 유효 사례 2,639건이다. 텍스트는 장애내용과 조치내용을 결합했고, 표기 통합과 불용어 제거 후 모델을 비교하였다.

| 모델 | 설정 | 주요 지표 | 해석 |
|---|---|---|---|
| TF-IDF 빈도분석 | 단어·부분 bigram | 주요 키워드: IE모드, 열람, 스캔, 프린터, 한글, 온나라 | 기초 현황용으로 좋지만 유형 자동분류는 어려움 |
| LDA | k=6,8,10,12 비교 | k=10 perplexity 257.7로 테스트 중 최저 | 전통적 토픽모델이라 설명은 좋지만 짧은 문장에서는 토픽이 섞임 |
| NMF | k=6,8,10,12 비교 | k=10 reconstruction error 46.885, 토픽 크기 116~408건 | 핵심어와 대표 사례가 선명해 보고서 본문 모델로 적합 |
| KMeans | SVD 50차원 후 k=6,8,10,12 비교 | k=12 cosine silhouette 0.190 | 강제 군집이라 대형 잡음 군집이 생김. 보조 비교용 |
| 유사도 그래프 | TF-IDF cosine threshold 0.70~0.95 | 0.85 기준 915건, 3,640쌍, 전체 34.7% | 반복 사례 분석에 매우 적합 |
| 룰 마이닝 | 장애내용 -> 조치내용 | CMS 접속 에러 -> IE모드 호환성 조치 25건, 신뢰도 0.86 | 표준 조치 매뉴얼 후보 추출에 적합 |
| z-score 이상탐지 | 일별 처리 건수 | 2025-10-13 40건, z=6.93 | 특정 업무량 급증일 설명에 적합 |

## 최종 추천 분석 프레임

보고서 제목은 `의왕시 전산장비 유지보수 데이터 기반 반복장애 및 표준조치 분석`이 가장 자연스럽다.

최종 모델 구성은 다음처럼 잡는 것을 추천한다.

1. 데이터 전처리: 빈 템플릿 행 제거, 날짜 보정, 표기 통합, 불용어 제거
2. 기초 현황: 월별·부서별·일별 유지보수 현황
3. 장애유형 모델링: TF-IDF + NMF 토픽모델
4. 모델 비교: LDA, NMF, KMeans, 유사도 그래프 비교
5. 반복사례 분석: 코사인 유사도 그래프와 Union-Find 그룹화
6. 표준조치 추출: 룰 마이닝 기반 장애-조치 매핑
7. 이상탐지: 일별 z-score 급증일 탐지
8. 운영 제언: 사전 안내, 표준 매뉴얼, 예방점검, 검색기 개선
